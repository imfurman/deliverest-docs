# Отчёты

Раздел «Отчёты» помогает управлять продажами, сервисом и финансами Deliverest. На стартовой странице собраны ссылки, которые можно сгруппировать под задачи конкретной команды и добавить в «Избранное».

## Как ориентироваться

- На странице `/admin/report` выберите нужную группу — например, «Продажи» или «Сервис». Структуру карточек можно настроить, чтобы вверху были ваши ежедневные отчёты.
- Каждый отчёт открывается в новой вкладке, сохраните фильтры и при необходимости добавьте страницу в закладки браузера.
- Экспорт доступен по кнопке в правом верхнем углу. Файл учитывает выбранные фильтры, поэтому перед выгрузкой стоит проверить диапазон дат, цех и дополнительные параметры.
- Отдельные служебные страницы (журнал фоновых задач, координаты курьеров) находятся в меню «Ещё».

## Основные категории

- [Стандартные отчёты](standard.md) — продажи, выручка, маркетинг, финансы. Здесь собраны «Цехи», «Продажи по позициям», «Заказы по платформам», «Скидки» и другие инструменты для оценки выручки и акций.
- [KPI и аналитика](kpi.md) — контроль SLA, скорости доставки и качества обслуживания. Включает «KPI | Скорость доставки (цеха)», «KPI | Скорость доставки (курьеры)», «Среднее время переходов между статусами», а также дашборды реального времени.
- [Оперативные показатели](operational.md) — сменное управление курьерами, операторами, производством и качеством. Подойдут «Водители по дням», «Принятые заказы операторов», «Загрузка цехов», «Отзывы клиентов», отчёт по отменам и журналы действий.

Эти страницы раскрывают отчёты детальнее: где их найти, какие вопросы они закрывают и как трактовать показатели.

## Дашборды аналитики

Для руководителей доступны визуальные панели:

- **Доходы** (`/analytics/incomeReport`) — динамика оборота, сравнение с отраслевыми значениями, прогноз месяца.
- **Сводный отчёт** (`/analytics/commonReport`) — заказов по сегментам лояльности, источники обращений, рейтинг клиентов и отзывы.
- **Консолидированный отчёт** (`/analytics/consolidatedReport`) — сравнение показателей компании с ближайшими конкурентами по обороту.

Дашборды используют те же фильтры, что и табличные отчёты, и позволяют выгружать данные прямо из виджетов.

## Практические советы

- Фильтры по датам поддерживают сменный режим: можно задать период «со смены до смены», чтобы сравнивать рабочих и выходных.
- Если отчёт нужно проверять каждый день, добавляйте ссылку в «Избранное», чтобы она отображалась на первой панели.
- Для расследования SLA сопоставляйте KPI-отчёты с данными «Рейсы и статусы (`trips_and_statuses`)» и «Заказы по цеху (`workshop_orders`)»: первый укажет на проблему, второй даст детали.
- При работе с акциями проверяйте пару «Скидки» + «Детализация скидок»: агрегированная статистика и конкретные заказы помогут оценить эффективность.
- В спорных финансовых ситуациях используйте дуэт «Долги между цехами» + «Детализация долгов по заказам» и выгрузку «Инкассация курьеров» — это ускоряет сверку инкассации и взаиморасчётов.
